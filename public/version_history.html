Detailed Version Timeline

v5.1 Vectorized Baseline

Initial engine — Vectorization + Adaptive Densification

Architecture

  • Single-file engine with inline haversine/scoring
  • Unidirectional A* on full OSMnx walk network
  • 6-component bidirectional scorer (forward + backward proximity)

Key Features

  • Uniform densification at configurable spacing
  • Adaptive densification: tighter on curves (based on turning angles)
  • Vectorized haversine using NumPy (no per-point Python loops)
  • Coarse grid search: rotation × scale × offset
  • Shape-aware routing via route_shape_aware()

Known Issues

  • No heading penalty — perpendicular roads freely used
  • No corridor pruning — A* searches entire graph (~12K nodes)
  • Occasional trunks from greedy waypoint snapping

v6.0 Segment-Constrained Tube Routing

Curvature-based densification + Tube corridor

Changes from v5.1

  • Segment-constrained tube routing: routes each shape segment independently inside a tube corridor
  • Curvature-based densification: more points inserted at high-curvature areas
  • GeoPandas spatial index (sindex): corridor pruning reduces graph to ~500 nodes
  • Adaptive tube radius: 100–500m based on node density
  • Multi-objective weight function: shape deviation + road type

Architecture Changes

  • Extracted routing.py — graph fetching, corridor building, A* routing
  • Extracted geometry.py — haversine, densification, sampling utilities
  • Extracted scoring.py — 6-component bidirectional scorer
  • Graph caching: memory + disk with MD5 keys

Known Issues

  • Occasional trunks from tube fragmentation in sparse areas
  • Still no heading penalty — perpendicular edges unconstrained
  • No U-turn control — 180° reversals anywhere on route

v7.0 Anchor-Point Routing

Anchor waypoint strategy + Shape-aware infill

Changes from v6.0

  • Anchor-point routing: identifies high-curvature vertices and routes skeleton first
  • Corrective infill: detects >80m deviations and inserts corrective waypoints
  • Shape deviation weight: custom weight function penalizing distance from ideal
  • OSRM fallback: when osmnx unavailable, routes via OSRM API

Architecture Changes

  • 3-tier fallback: anchors → shape-aware → basic graph routing → OSRM
  • Anchor identification via turning-angle threshold (>30°)
  • Refinement pass: re-route worst-deviation segments

The Problem That Emerged

  • "Trunk" spikes: hard waypoint snapping forces out-and-back detours
  • Greedy anchor infill creates perpendicular spikes when no through-road exists
  • No heading penalty → router cheerfully uses perpendicular roads
  • No U-turn control → free 180° reversals produce visible artifacts

v8.0 Soft-Constraint Field (No-Trunk)

Complete architectural refactor — CoreRouter + Fréchet scoring

Major Changes from v7.0

  • CoreRouter class: unified routing pipeline with soft-constraint edge weighting
  • 5-component cost function: C = w_net·network + w_prox·proximity + w_head·heading + w_uturn·uturn + w_bio·biomech
  • Heading penalty: edges deviating >20° from local tangent are penalized (0–300m)
  • Apex sensitivity: U-turns only allowed within 15m of sharp vertices (bottom point, top dip)
  • Bidirectional A*: searches from both directions simultaneously (2× speedup)
  • Vectorized edge weights: one NumPy pass precomputes all edge costs (5-10× vs per-call)

Scoring Revolution

  • Discrete Fréchet distance: "dog-walker" metric replaces Hausdorff
  • Scale-invariant normalization: both polylines normalized to 1×1 box
  • Composite v8 score: 40% Fréchet + 20% coverage + 15% perpendicular + 15% heading + 10% length-ratio

New Files

  • core_router.py — CoreRouter, heading penalty, U-turn mask, bidir A*, cycle finder
  • scoring_v8.py — discrete Fréchet, composite v8 score
  • geometry.py — tangent field, apex detection, normalize_to_unit_box

Cost Formula

C(u,v) = 1.0·C_network + 12.0·C_proximity + 8.0·C_heading + 1.0·C_uturn + 2.0·C_biomech

Remaining Issues

  • Additive cost formula: proximity penalty can be overcome by short edges
  • No waypoint skip logic — all sampled waypoints are force-routed
  • Heading cap at 300m — perpendicular edges still sometimes cheaper than alternative
  • No Fréchet reject gate — poor routes pass silently

v8.1 Flow-Aware Router (Trunk-Killer)

March 2026 — Multiplicative penalty field + Skip logic + Tangent pruning

Root Cause Analysis

  • v8.0's additive cost let short perpendicular edges "sneak through" the penalty
  • Every sampled waypoint was force-routed — no mechanism to skip bad detours
  • Heading penalty capped at 300m — still cheaper than some alternatives

AGENT 'GRAPH' — Routing Engine Changes

  • Multiplicative cost function: C = L × H × (1 + β·d²) + heading + uturn
  • Trunk-Killer skip logic: evaluates detour/direct cost ratio for each waypoint; skips if ratio > 3×
  • Heading penalty cap raised: 300m → 500m for perpendicular edges, with new 350m intermediate band
  • U-turn penalty raised: 2000m → 5000m for non-apex reversals
  • β = 0.0003: 100m deviation → 4× base cost; 200m → 13× base cost

AGENT 'SPATIAL' — Pre-processor Changes

  • Tangent pruning: removes edges deviating >35° from local ideal-line tangent (within 150m)
  • Connectivity-safe pruning: checks weak connectivity after pruning; reverts if too much damage
  • Iterative corridor widening: 100m steps up to 800m (was single jump to max_radius)

AGENT 'VISION' — Quality Gatekeeper

  • Fréchet reject gate: normalized Fréchet > 0.12 → score doubled → forces optimizer to pick different candidate
  • frechet_normalized(): new function returning unitless [0,1] Fréchet for gating
  • Shapely fast-path: tries shapely.hausdorff_distance (C-level) before NumPy fallback

New Cost Formula

C(u,v) = L_edge × H_mult × (1 + 0.0003 × d_perp²) + 8.0 × heading_penalty + uturn_penalty

Where d_perp = perpendicular distance to ideal centerline (metres)
At d=100m: multiplier = 4.0× | At d=200m: multiplier = 13.0× | At d=300m: multiplier = 28.0×

Skip Logic Detail

For each waypoint W_i:
  detour_cost = route_cost(W_{i-1} → W_i) + route_cost(W_i → W_{i+1})
  direct_cost = route_cost(W_{i-1} → W_{i+1})
  IF detour_cost / direct_cost > 3.0 THEN skip W_i
  Max skips capped at 30% of waypoints

v8.1-abs Symmetry-First Router

March 2026 — Bilateral Symmetry + Cusp-Apex Verticality + Curvature Guard + Mirror-Search

Philosophy

  • Treats bilateral symmetry as a hard constraint — trunks are mathematically impossible
  • Routes maximize "Heart-ness" via mirror-error, not Euclidean proximity
  • SymmetryScorer: 6-component scoring (mirror + verticality + curvature + trunk + TA-Fréchet + spatial)
  • Cusp-to-Apex constraint: both lobes originate from cusp (top) and terminate at apex (bottom)
  • Curvature guard: prevents flattening into rectangles by comparing curvature profiles

AGENT 'SPATIAL' — Landmark Extraction

  • Cusp detection: top indentation (second-sharpest turn, separated from apex)
  • Apex detection: bottom sharp point (sharpest turn in template)
  • Symmetry axis: unit vector from Cusp to Apex for reflection
  • Lobe-Arc zones: max-curvature regions on left and right halves

AGENT 'GRAPH' — Symmetry-Weighted Router

  • Symmetry cost: W = L × H_w × (1 + β·S_err + γ·θ_dev)
  • Mirror-Search: routes left lobe, penalizes used edges 50×, routes right lobe
  • Cycle-Search: nx.minimum_cycle_basis for symmetric loop discovery
  • Apex-aware turns: 180° allowed at apex, max penalty in lobes
  • Tangent pruning: removes edges >35° from local tangent

AGENT 'VISION' — SymmetryScorer (6-component)

  • Mirror Error (25%): Fréchet between left lobe and reflected right lobe
  • Verticality (15%): cusp-to-apex axis alignment + origin/terminus pinning
  • Curvature (15%): flatness detection, distribution mismatch, lobe-arc peaks
  • Trunk Penalty (15%): detects out-and-back spikes × 10
  • Turning-angle Fréchet (15%): shape grammar fidelity
  • Spatial Fréchet (15%): geometric grounding

Cost Formula

W(u,v) = L × H_w × (1 + β·S_err + γ·θ_dev)
β = 0.005 per-metre symmetry penalty, γ = 1.5 heading weight
S_err = mirror distance (edge midpoint reflection → nearest ideal point)
Apex bonus: U-turns cheap within 30m; lobe reversals +500m
